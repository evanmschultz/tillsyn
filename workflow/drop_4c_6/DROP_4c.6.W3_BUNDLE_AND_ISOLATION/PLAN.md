### Decomposition shape — six atomic droplets across three packages, mostly serialized on `internal/app/dispatcher/cli_claude/render` package compile

The six droplets fall into three package-disjoint groups:

1. **`internal/app/dispatcher` package** — D1 (`SystemPromptTemplatePath` plumbing through `BindingResolved` + `ResolveBinding`). Edits `cli_adapter.go` + `binding_resolved.go` + their `_test.go` siblings.
2. **`internal/app/dispatcher/cli_claude/render` package** — D2 (3-tier resolver), D3 (frontmatter strip wiring), D5 (post-render validator + sentinel test), D6 (doc-comment correction). All four edit `render.go` and `render_test.go` — same compile unit, MUST serialize per `~/.claude/agents/go-planning-agent.md` § "Cascade Design — Atomic Droplets + Parallelization (HARD RULES)": "Siblings sharing a `packages` entry (even with disjoint paths within that package) → `blocked_by`."
3. **`internal/app/dispatcher/cli_claude` package** — D4 (defense-in-depth env vars in `env.go`). Edits `env.go` + `adapter_test.go`. File-disjoint from D2/D3/D5/D6 (parent package, not child); package-disjoint too (`cli_claude` vs `cli_claude/render`). Parallels D1.

Cross-group dependencies:

- **D2 reads `binding.SystemPromptTemplatePath`** — D2 `blocked_by` D1.
- **D3 mutates the body D2 produces** — D3 `blocked_by` D2.
- **D5 validates the body D2 + D3 produce** — D5 `blocked_by` D2, D3.
- **D6 edits the doc-comment block adjacent to (and inside the function-doc-comment range of) the `assembleAgentFileBody` function D2 rewrites** — D6 `blocked_by` D2, D3, D5. Even though D6 is doc-only, it edits the same `render.go` file (same package compile lock); serializing D6 last avoids any chance of D2/D3/D5's function-body edits clobbering D6's adjacent doc-comment edit.
- **D4 is fully parallel** — different package (`cli_claude` vs `cli_claude/render` vs `dispatcher`), different file (`env.go` vs `cli_adapter.go` / `render.go`).

The render-package serial chain therefore reads:

```
W3.D1 ──▶ W3.D2 ──▶ W3.D3 ──▶ W3.D5 ──▶ W3.D6
              (D4 parallel — own package, own file)
```

Tests are co-located with each droplet's production file. D1 appends to `cli_adapter_test.go` + `binding_resolved_test.go`; D2/D3/D5 append to `render_test.go`; D6 is doc-only (no test changes); D4 appends to `adapter_test.go`.
