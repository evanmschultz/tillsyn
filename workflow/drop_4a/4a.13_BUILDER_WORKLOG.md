## 3. "To Do" vs "Todo" Disambiguation (REQUIRED PER DROPLET SPEC)

The REVISION_BRIEF wording lists the canonical state IDs as: `Todo / In Progress / Complete / Failed / Archived`.

The actual seeded **display name** at HEAD is `"To Do"` (with a space). This is **intentional and correct**:

- `defaultStateTemplates` (`service.go:2305`) seeds `Name: "To Do"` (display name) with `ID: "todo"` (canonical state-ID slug).
- `normalizeStateID` (`service.go:2350-2398`) maps the display name `"To Do"` → canonical `"todo"`:
  - lowercase, then strip non-`[a-z0-9]` and collapse to `_` → `"to_do"`.
  - The function then re-canonicalizes `"to_do"` to `"todo"` (the documented "to-do remains kebab-spelled canonical" rule per the doc-comment at `:2356`).
- The `"To Do"` display-name spelling is **UI-friendly text**; the canonical state-ID stays `todo`.

**This droplet does NOT change the display-name spelling.** `"To Do"` (with space) is the correct, intentional display name. The REVISION_BRIEF's `Todo` reference describes the canonical state-ID, not the display name. There is no drift here, and no QA Falsification finding should flag the wording.
