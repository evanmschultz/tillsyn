## Counterexamples Summary

**CONFIRMED (documentation drift — no code-correctness impact):**

- **A13.** `cleanup.go` lines 20–22 misattribute the auth-revoke wiring as "Wave 3 of Drop 4a (the spawn-redesign work)." Wave 3 of Drop 4a = auth integration (approve op); spawn-redesign = Drop 4c F.7 (`workflow/drop_4c/SKETCH.md` line 123). Trivial copy-edit fix.
- **A14.** `cleanup.go` lines 128–131 + 159–162 doc-comments claim `errInvalidCleanupDep` "wraps ErrInvalidDispatcherConfig" — implementation uses `errors.New(...)`, no wrap. Callers using `errors.Is(err, ErrInvalidDispatcherConfig)` will NOT match cleanup misconfiguration. Worklog #6 acknowledges the wrap-vs-string tradeoff but the doc-comment still lies. Either flip the doc to match code OR flip the code to match doc (one-line change to `fmt.Errorf("%w: %s is nil", ErrInvalidDispatcherConfig, name)`).

**ACCEPTED (spec-aligned, not counterexamples):**

- **A2** (unbounded `cleaned` map): worklog + cleanup.go open acknowledgment, Drop 4b plan.
- **A6** (panic-resilience absent): outside spec scope; over-engineering would be the bug.
- **A9** (no production `monitorUnsubscriber` today): explicit 4a.23 deferral.
- **A10 observation** (no test pins constructor's wiring of `revokeAuthBundleStub`): coverage nit, not a correctness defect.
