### 1.5 Attack: NULL / "null" / empty `paths_json` decode surprise — REFUTED

**Hypothesis.** `TEXT NOT NULL DEFAULT '[]'` should prevent NULL from disk, but `json.Unmarshal` of a string `"null"` would set the slice to `nil`. Test gap?

**Evidence.** `decodeActionItemPathsJSON` (`internal/adapters/storage/sqlite/repo.go:3247-3257`) explicitly handles three edge cases BEFORE `json.Unmarshal`:
```go
trimmed := strings.TrimSpace(raw)
if trimmed == "" || trimmed == "null" {
    return nil, nil
}
```
- empty raw → nil.
- "null" raw → nil.
- "[]" raw → falls through to `json.Unmarshal` which produces a non-nil empty `[]string{}`.

The decoder is symmetric with the encoder (`encodeActionItemPathsJSON` returns `"[]"` for nil/empty input). Round-trip preserved.

**Counterexample status.** REFUTED. Defensive decoding handles the edge cases the schema's NOT NULL DEFAULT would otherwise mask.

### 1.12 Attack: Spec deviation on Update pointer-sentinel — CONFIRMED NIT (not FAIL)

**Hypothesis.** WAVE_1_PLAN.md §1.1 line 70 explicitly says: `UpdateActionItemInput struct gains Paths []string (no pointer-sentinel — empty slice is the zero value and is meaningful)`. Builder shipped `*[]string` instead. This is a spec deviation.

**Evidence.**
- `internal/app/service.go:584` — `Paths *[]string` (pointer-sentinel).
- `internal/adapters/server/common/mcp_surface.go:171` — `Paths *[]string` (pointer-sentinel).
- `internal/adapters/server/mcpapi/extended_tools.go:876` — args struct uses `Paths *[]string` (pointer-sentinel).
- All three diverge from WAVE_1_PLAN.md's "no pointer-sentinel" rule.

**Counterexample analysis.**
- Engineering correctness: the builder's pointer-sentinel choice MATCHES the Drop 3.21 precedent (Owner/DropNumber/Persistent/DevGated all use pointer-sentinel on Update for the same clobber-protection reason). The WAVE_1_PLAN authoring missed this — using value-type `[]string` on Update would mean a description-only `till.action_item update` call (with `paths` omitted from JSON) silently clobbers a planner-set `Paths` declaration to nil. That's exactly the failure mode pointer-sentinel exists to prevent.
- The builder's inline doc at `service.go:584-590` documents the rationale: *"a description-only update by an agent must NOT silently clobber a planner-set Paths declaration."* This matches the 3.21 reasoning verbatim.
- No worklog file for 4a.5 exists in `workflow/drop_4a/` — the deviation is not recorded as a decision-log entry.

**Verdict on this attack.** CONFIRMED as a process NIT, NOT a code-failure counterexample. The engineering is correct; the plan was buggy. Demoting from FAIL → NIT because:
1. The fix prevents a real silent-clobber bug.
2. The shape matches a strong precedent (Drop 3.21 four-field expansion).
3. `mage ci` green; round-trip tests cover the behavior.
4. Future droplets (4a.6 packages, 4a.7 files) will likely repeat the deviation — the WAVE_1_PLAN wording should be amended (or a meta-decision logged) before they spawn.

**Counterexample status.** CONFIRMED — but as a NIT (process: missing decision-log entry for plan deviation). Not a verdict-flipping counterexample.
