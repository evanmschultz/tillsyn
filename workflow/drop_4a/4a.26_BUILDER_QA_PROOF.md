2. **Bypass-via-ResolvedByType attack** — Could a caller pass `ActorTypeUser` to skip the assertion? Yes by design — the plan explicitly exempts user (dev-TUI) approvals. The W3.1 adapter-layer pre-check rejects empty `agent_instance_id` / `lease_token` BEFORE this method is reached for the agent path, so the only way to legitimately reach this method with `ActorTypeUser` is the dev-TUI flow, which has no agent-instance / lease to provide. Mitigated.
3. **Side-effect-before-validation attack** — Does the assertion fire before state transition? Yes — `:608-612` is the validation gate, `:614` (`r.State = AuthRequestStateApproved`) is the first mutating side-effect. Mitigated.
4. **ALTER idempotency on existing schema** — Does `isDuplicateColumnErr` actually catch SQLite's error? `modernc.org/sqlite` returns "duplicate column name" verbatim for an ADD COLUMN of an existing column; `:944-948` lowercases the error string and substring-matches. The idempotency test runs `ensureAuthRequestSchema` twice and asserts nil — direct proof. Mitigated.
5. **JSON `omitempty` elision attack** — Could populated empty-strings still emit keys? `omitempty` on string fields elides on empty-string. The test verifies BOTH directions: populated-keys-present (orch path) AND keys-absent (dev-TUI path). Mitigated.
6. **Worklog count drift** — Worklog claims +9 net passing; 6+2+1=9 matches live counts. Verified.

No unmitigated counterexamples found.
