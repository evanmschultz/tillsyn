## 3. Design Decisions

### 3.1 Internal `actionItemReader` interface

The spec mandates `NewDispatcher(svc *app.Service, broker app.LiveWaitBroker, opts Options) (*dispatcher, error)`. To exercise `RunOnce` skip paths in tests without standing up a full `Service` + `Repository` graph, the dispatcher stores the service via a narrow consumer-side interface:

```go
type actionItemReader interface {
    GetActionItem(ctx context.Context, actionItemID string) (domain.ActionItem, error)
}
```

`*app.Service` satisfies this (the method exists at `internal/app/auth_scope.go:21`). Tests construct `*dispatcher` directly through the unexported struct literal with a `stubActionItemReader`. Production code only goes through `NewDispatcher`.

This is idiomatic Go (consumer-side interfaces) and pays dividends for Wave 2.5 (walker tests will reuse the same stub pattern).

### 3.2 `Options` is intentionally empty

Per spec: forward-compat open struct. Wave 2.1 leaves it at zero value. Added a placeholder `_` field so positional struct-literal callers fail loudly when later droplets add real fields.

### 3.3 Skip semantics for the skeleton

Wave 2.1 has no walker / locks / spawn yet. The 2.1 contract:

- Empty/whitespace ID → `ResultSkipped`, no service call.
- `app.ErrNotFound` from service → `ResultSkipped`.
- Non-todo lifecycle state → `ResultSkipped`.
- Todo state → `ResultSkipped` (skeleton stops here; later droplets add the spawn path).
- Any other service error → bubbled up wrapped with `%w`.

Pinned by `TestRunOnceTodoActionItemSkipsForNow` so a future droplet flipping todo → spawn is a deliberate test edit, not a silent regression.

### 3.4 `Start` / `Stop` return `ErrNotImplemented`

Spec allows this for Wave 2 — Drop 4b wires the continuous-mode loop. Sentinel error `ErrNotImplemented` is exported so callers can `errors.Is` against it.

### 3.5 Compile-time interface assertion

`var _ Dispatcher = (*dispatcher)(nil)` at the bottom of `dispatcher.go` catches any future drift where a method goes missing. Test `TestNewDispatcherSucceedsWithValidArgs` runtime-asserts the same thing.

### 3.6 Clock injection

`dispatcher.clock func() time.Time` defaults to `time.Now` in `NewDispatcher` and is settable via the unexported struct literal in tests. `TestRunOnceUsesInjectedClock` proves the wiring.
