## Wave Purpose

Wave 3 ships the auth-approval-cascade primitive (Drop 1.6 absorbed): orchs may approve auth requests for non-orch subagents within their own subtree, removing the dev TUI bottleneck that today's S2 dev-fallback paragraph documents. The dispatcher's auto-spawn loop landed in Wave 2 is the only consumer — Drop 1.6 was deferred from standalone shipping precisely to land alongside its consumer here. Wave 3 ships the **binary capability**: orch-self-approves-non-orch-subagent-in-subtree, full-stop. Configurability knobs (per-role thresholds, rate limits, etc.) are **explicitly out of scope** per brief item 3 — they remain deferred to a future refinement drop.

Five droplets:

1. **W3.1** lands the auth-layer rule + role-gate + no-configurability constraint (bundled per brief permission).
2. **W3.2** lands the dev opt-out toggle on `ProjectMetadata`.
3. **W3.3** lands the audit trail (approving orch's `agent_instance_id` + `lease_token` + `principal_id` recorded on every approved auth request).
4. **W3.4** lands the MCP-layer test coverage (4 cases per brief item 6).
5. **W3.5** lands the prompt + memory updates (deletes the S2 dev-fallback paragraphs + memory caveats).
