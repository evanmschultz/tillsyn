## Design Choices

- Mirrored `fileLockManager` (`locks_file.go`, committed `e3302a3`) verbatim — same `sync.Mutex` + dual-map shape (`holders map[string]string` for the package→holder index, `itemPackages map[string]map[string]struct{}` as the derived per-item index for O(holder-set) Release). Only the variable rename (`paths` → `packages`, `itemPaths` → `itemPackages`) and the doc comment differ between the two managers.
- **Independence call-out** — the `locks_package.go` package-level doc comment explicitly states the manager is INDEPENDENT of `fileLockManager`: "A path lock on `internal/app/foo.go` and a package lock on `internal/app` do NOT collide inside either manager." Cross-locking is owned by walker (4a.18) + conflict detector (4a.20); locking it here would smear the policy across two components.
- **No premature generic** — kept `packageLockManager` and `fileLockManager` as two distinct types per WAVE_2_PLAN §2.4 line 151 and PLAN.md row 4a.17. Drop 4b plans diverge: package-lock will grow `go list -json` resolution, file-lock stays opaque.
- **API parity** — `Acquire(actionItemID string, packages []string) (acquired []string, conflicts map[string]string, err error)` + `Release(actionItemID string)` is identical in signature shape to `fileLockManager`. Empty-input early return, idempotent same-holder reacquire, zero-value lazy-init, opacity (no normalization), partial-acquire semantics — all preserved verbatim.
- **Error return retained** — `err` is always nil today; signature exists so Drop 4b's optional SQLite mirror can surface persistence failures without an API break.

| 7 | `TestPackageLockEmptyInputsAreNoOps` | supporting | nil/empty packages early-return; Release on unknown ID; idempotent reacquire |

| 9 | `TestPackageLockIndependentFromFileLock` | supporting | the doc-comment independence guarantee — locking `"internal/app"` in fileLockManager does NOT pre-lock the same string in packageLockManager (and vice versa) |

Test 9 is the runtime teeth on the doc-comment independence claim — without it the claim is just prose.
