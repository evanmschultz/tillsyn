### 1.2 `FormatPath(path string)` — scoped formatter

**Spec:** "preserves the prior single-path / single-directory semantics: trim, reject empty, `os.Stat` check, `runGofumptWrite([]string{path})`."

**Evidence:** `magefile.go:212-222`

```go
// FormatPath rewrites one file or directory with `go tool gofumpt -w`. Use Format for the whole tracked tree.
func FormatPath(path string) error {
    path = strings.TrimSpace(path)
    if path == "" {
        return errors.New("path is required")
    }
    if _, err := os.Stat(path); err != nil {
        return fmt.Errorf("gofumpt path %q: %w", path, err)
    }
    return runGofumptWrite([]string{path})
}
```

**Conclusion:** PASS. Trim (line 214), reject empty (lines 215-217), `os.Stat` check with wrapped error (lines 218-220), `runGofumptWrite([]string{path})` (line 221).

**Behavior delta from pre-edit:** previously `path == "."` was an alias for whole-tree; now `"."` would pass `os.Stat` and reach `runGofumptWrite([]string{"."})`. That's intentional and consistent with the split — whole-tree is `Format()`, not `FormatPath(".")`. Acceptable.

## 3. Optional `format-path` Alias — Rationale Assessment

The spec lists `"format-path": FormatPath` as **optional** ("if the alias dispatch supports it; otherwise document `mage format-path ./path` as the explicit CLI form — builder picks based on mage v1.17 behaviour"). The builder chose to add it.

**Worklog:** A builder worklog at `workflow/drop_4a/4a.1_BUILDER_WORKLOG.md` was NOT found at QA time. This is a small process gap (the builder did not leave a written rationale alongside the work), but the alias choice is independently defensible:

- Without the alias, the camel-case Go identifier `FormatPath` is exposed by mage as kebab-case `format-path` only via the alias — if you skip the alias, the CLI form becomes `mage formatPath` (camel-case fallback per mage's discovery), which breaks the project's existing kebab-case convention (`test-pkg`, `test-func`, `test-golden`, `test-golden-update`).
- The alias is symmetric with `"format-check": FormatCheck` (also added in this droplet) and consistent with prior precedent (`"test-pkg": TestPkg`, `"test-func": TestFunc`).
- Zero risk: the alias only adds a CLI-name → identifier mapping. No collision with existing entries.

**Verdict:** the alias addition is sound and improves CLI ergonomics. Not a finding.

**Process nit (not a verdict-changing finding):** the missing `4a.1_BUILDER_WORKLOG.md` should be authored before the droplet is fully closed, per the WORKFLOW.md per-droplet artifact convention. The orchestrator may want to ask the builder to drop a short worklog (purpose, decisions taken, gate output) for audit-trail completeness alongside the QA twins.
