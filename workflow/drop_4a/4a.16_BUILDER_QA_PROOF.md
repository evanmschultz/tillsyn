### AC-6 — Deadlock-free; flat synchronous API documented

**Verdict: PROVEN.**

Documentation evidence: `locks_file.go:31–34` struct doc comment — "Deadlock-free contract: the API is flat and synchronous. There are no callbacks, no goroutines spawned by the manager, and no nested locking. Acquire and Release run to completion before returning; a caller cannot re-enter the manager from inside one of its calls."

Code evidence:

- `Acquire` body (`locks_file.go:82–117`): single mutex acquisition (`m.mu.Lock()` line 88, `defer m.mu.Unlock()` line 89). No callbacks, no goroutines spawned, no method calls on `m` inside the locked region.
- `Release` body (`locks_file.go:127–139`): single mutex acquisition (line 128, defer 129). No callbacks, no goroutines spawned, no nested `m.<method>` calls.

The mutex is held at most once per call; no method calls another method that re-acquires the mutex; no caller-supplied function is invoked under the lock. Re-entrancy via the same goroutine is impossible because Go's `sync.Mutex` is non-reentrant — but the code design also doesn't try to call back into itself, so the deadlock surface is zero.

The `TestFileLockConcurrentAcquireRaceFree` 32-goroutine race test would deadlock or detect a race if any of these invariants were violated; it passes clean under `-race`.

## Cross-Spec Sanity

- **`paths` order** — spec says nothing about ordering; implementation preserves input order (`locks_file.go:114`, `acquired = append(acquired, path)` inside the `range paths` loop). `TestFileLockAcquireSinglePathSucceeds` asserts a single-element ordering trivially; multi-path tests use order-insensitive `equalStringSlices` helper. No drift.
- **Empty `actionItemID`** — spec says nothing; implementation treats it opaquely (no validation). `Acquire("", [a])` would lock `a` to the empty-string holder; `Release("")` would un-lock paths held by the empty-string holder. The doc at `locks_file.go:71–73` documents this as "treated as opaque (no validation in this droplet)." Acceptable as documented.
- **`err` always nil** — spec says nothing; implementation always returns `nil`. The doc at `locks_file.go:79–81` names the rationale (Drop 4b SQLite-mirror reservation). Forward-compatible signature.
