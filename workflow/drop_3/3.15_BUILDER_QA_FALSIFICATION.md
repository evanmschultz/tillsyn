## 1. Counterexamples

### CE1 — Stale `till kind upsert` example in `cmd/till/help.go` overrides correct inline help [CONFIRMED, MINOR]

**Premise.** The droplet acceptance (PLAN.md droplet 3.15 finding 5.B.13 / CE8) requires DELETE of the `till kind upsert` CLI subcommand, with deprecation documented. Read-only `till kind list` + `till kind allowlist *` may stay.

**Evidence.**

- `cmd/till/main.go:887-916` — `kindCmd` carries inline `Long`/`Example` referencing only `till kind list` + `till kind allowlist list ...`. `kindCmd.AddCommand(kindListCmd, kindAllowlistCmd)` at `:972` confirms the upsert subcommand is no longer wired.
- `cmd/till/help.go:99-108` — `commandHelpSpecs["till kind"]` still carries the stale example list:
  ```go
  Example: []string{
      "  till kind list",
      "  till kind upsert --id research-actionItem --display-name \"Research ActionItem\" --applies-to actionItem",
      "  till kind allowlist list --project-id PROJECT_ID",
      "  till kind allowlist set --project-id PROJECT_ID --kind-id actionItem --kind-id research-actionItem",
  },
  ```
- `cmd/till/help.go:124-141` — `commandHelpSpecs["till kind upsert"]` is a complete dead entry (full Long + 3 examples) including the legacy `--template-json` flag and `--allowed-parent-scopes` flag, both of which were deleted in this droplet.
- `cmd/till/main.go:1726` — `applyCommandHelp(rootCmd)` runs AFTER `kindCmd` is added; `applyCommandHelpSpecs` (`help.go:393-406`) walks the tree and OVERWRITES `cmd.Long` and `cmd.Example` from `commandHelpSpecs` when keyed by `cmd.CommandPath()`. So `till kind --help` will display the stale list, not the freshly-set inline `Example`.
- The `"till kind allowlist set"` example on line 107 also references `--kind-id actionItem --kind-id research-actionItem` — kind names from the Drop-1.75-collapsed vocabulary that are themselves stale (no longer valid kinds).

**Trace.**

1. User runs `till kind --help`.
2. Cobra reads `cmd.Long` and `cmd.Example` of the `kind` command.
3. Both fields were overridden by `applyCommandHelpSpecs` at startup using `commandHelpSpecs["till kind"]`.
4. The output contains a `till kind upsert ...` line that no longer exists as a subcommand — running it yields a Cobra "unknown command" error, contradicting the help that just printed it.
5. The orphan `commandHelpSpecs["till kind upsert"]` entry is unreachable (path not registered) but still ships in the binary; future planners or `grep` audits will trip on it.

**Conclusion.** CONFIRMED counterexample. The CLI surface is functionally deleted (the cobra subcommand IS gone), but the help text is the user-facing contract and it still advertises the deleted command and stale flags (`--template-json`, `--allowed-parent-scopes`, kinds `actionItem` / `research-actionItem`). Severity: minor user-visible drift; functional regressions are zero (running the suggested example produces a clean error).

**Suggested fix.** Edit `cmd/till/help.go`:

- Lines 103-108: drop the upsert example, keep only `till kind list` + `till kind allowlist list/set` examples (sync with main.go line 893-896 inline list).
- Lines 124-141: delete the entire `"till kind upsert"` map entry.
- Line 107: replace `--kind-id actionItem --kind-id research-actionItem` with closed-12-kind values (e.g. `--kind-id build --kind-id plan`).

These three edits are localized, do not touch the cobra tree, and bring help text in sync with the wire surface. Recommend a follow-up commit before drop close (or roll into 3.16 if scope is constrained).

**Unknowns.** None — the path is mechanical text editing.
