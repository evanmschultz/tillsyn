- `drop_number INTEGER NOT NULL DEFAULT 0`

### 2.4 scanActionItem arg-order mismatch — REFUTED

`scanActionItem` (`:2722-2789`) declares 4 new locals before the Scan call: `ownerRaw string`, `dropNumberRaw int`, `persistentRaw int`, `devGatedRaw int` (`:2749-2752`). ### 2.5 Index coverage — REFUTED

Two new indexes (`:442-443`):
- `idx_action_items_drop_number ON action_items(project_id, drop_number, owner)`
- `idx_action_items_owner_title ON action_items(project_id, owner, title)`

The auto-generator queries cited by the contract are:
1. `WHERE project_id = ? AND drop_number = ?` — covered by `idx_action_items_drop_number(project_id, drop_number, owner)` as a 2-column prefix scan.
2. `WHERE project_id = ? AND owner = ? AND title = ?` — covered by `idx_action_items_owner_title(project_id, owner, title)` as a full 3-column index match.

`TestRepository_IndexCoversDropNumberQuery` (`repo_test.go`) seeds 3 rows in drop 3, 2 in drop 4, 1 in drop 0; queries by `(project_id, drop_number=3)`; asserts 3 rows returned in id-sorted order. Query is correctness-tested, not EXPLAIN-plan tested — the comment notes this and explains why (SQLite EXPLAIN output is brittle across versions). Acceptable.

The second query path (`owner+title` lookup) is NOT directly exercised by a behavioral test, but the index exists and the same correctness pattern would apply. Borderline — minor coverage gap, NOT a counterexample.

### 2.7 Legacy fixture update scope (out-of-paths) — REFUTED

`internal/adapters/storage/sqlite/embedding_lifecycle_adapter_test.go` is OUT of 3.18's declared paths (paths only allow `repo.go` + `repo_test.go`). The diff shows ONLY 2 lines added — `owner` + `drop_number` columns to the legacy DB simulator's CREATE TABLE.

Necessity check: the test's purpose is to simulate a legacy DB without the modern schema, then call `Open()` which runs `migrate()` against that legacy DB. After 3.18, `migrate()` runs `CREATE INDEX IF NOT EXISTS idx_action_items_drop_number ON action_items(project_id, drop_number, owner)` and `CREATE INDEX IF NOT EXISTS idx_action_items_owner_title ON action_items(project_id, owner, title)`. SQLite rejects `CREATE INDEX` on non-existent columns. Without adding `owner` + `drop_number` to the legacy fixture, those CREATE INDEX statements would fail and `migrate()` would return an error. The fix is the minimum required for `migrate()` to complete on the legacy schema.

Why NOT add `persistent` + `dev_gated` to the legacy fixture too? Because neither column appears in any index, in any UPDATE/INSERT issued during `migrate()`, or in any SELECT issued during `migrate()`. The legacy fixture only needs columns that the migration SQL touches. `persistent` + `dev_gated` are only touched by `Repository.CreateActionItem` / `UpdateActionItem` / scanActionItem-using SELECTs, none of which fire during the legacy schema test (the test only opens the DB, never inserts an action item).

Verified empirically: `mage test-func ./internal/adapters/storage/sqlite TestRepositoryEmbeddingLifecycleSchemaMigratesLegacyDatabase` passes 1/1.

This is a minimal, load-bearing out-of-paths edit. Acceptable per the task brief explicitly allowing it.

### 2.8 Persistent/DevGated bool round-trip — REFUTED

`boolToInt` (`:3156-3161`): `true → 1`, `false → 0`. Storage column is INTEGER. Scan reads into `int`. Post-scan: `t.Persistent = persistentRaw != 0`. Round-trip preserves both states. Test cases include `persistent: false, devGated: false` (case 1), `persistent: true, devGated: true` (case 2), and mixed (cases 3, 4). All four bool-boundary states exercised. Reassign-on-update flow (`UpdateActionItem` from false→true) also exercised.

### 2.9 DropNumber overflow — REFUTED

`DropNumber int` in domain → SQLite `INTEGER` (64-bit) → scan into `int` (64-bit on every supported platform). No overflow path constructible for any realistic cascade drop number. Negative values reject in `domain.NewActionItem` (`internal/domain/action_item.go:258-260`) with `ErrInvalidDropNumber`. Test exercises `dropNumber=0`, `5`, and (after Update) `7` — all valid.

### 2.10 Owner empty-string round-trip — REFUTED

Domain trims surrounding whitespace via `strings.TrimSpace` (`:255`); empty stays empty (no default substitution). SQLite stores `''`. Scan reads back `''`. Test case `t-own-empty-zero` (`owner: ""`) exercises this path — assertion `loaded.Owner != tc.owner` would fire if any default substitution happened.
