### 1.1 NIT — `TestActionItemMCPRejectsEmptyOrInvalidStructuralType` lacks wire happy-path / preservation parity with Role

**Severity:** NIT (does not violate droplet contract).

The new test exercises three scenarios:

1. Empty `structural_type` rejected on create via the real adapter chain (`adapter.CreateActionItem` → `app.Service` → `domain.NewActionItem`).
2. Unknown `structural_type` rejected on create via MCP wire + stub.
3. Unknown `structural_type` rejected on update via MCP wire + stub.

The Role precedent at `internal/adapters/server/mcpapi/extended_tools_test.go:3450-3559` (`TestHandlerExpandedActionItemRoleRoundTrip`) covers five scenarios: create-with-valid-plumbs, create-without-empty-roundtrips, update-with-valid-plumbs, update-without-empty-preserves, create-with-invalid-rejected. The new StructuralType test omits the three positive-path / preservation scenarios.

Why this is NOT a counterexample:

- The droplet contract (PLAN.md §3.4) explicitly specifies: *"New `TestActionItemMCPRejectsEmptyOrInvalidStructuralType` exercises both empty (rejected on create) and unknown value (rejected on both create + update)."* The test does exactly that.
- Happy-path round-trip is exercised at the SQLite layer in droplet 3.3 (`internal/adapters/storage/sqlite/repo_test.go:2310-2435`).
- Cross-package fixture migration (~44 callsites) routes valid `structural_type` through the full chain in production-shaped tests; if the wire path were broken, `mage ci` would fail.
- Empty-on-update preservation through the wire is implicitly covered by the existing 9 update-via-wire tests that omit `structural_type` and pass under the current build.

A future hardening drop should mirror the Role precedent's five sub-tests for StructuralType so the wire happy-path becomes a first-class assertion rather than an inferred property. Not blocking for 3.4.

### 2.4 MCP `mcp.Enum` constraint at JSON-schema layer attack — REFUTED

`go doc -src github.com/mark3labs/mcp-go/mcp.Enum` confirms `mcp.Enum` only sets the JSON-schema `enum` field on the property and does NOT enforce server-side: *`func Enum(values ...string) PropertyOption { return func(schema map[string]any) { schema["enum"] = values } }`*. The constraint is advisory to MCP clients (e.g., the LLM). The wire path's `"not-a-structural-type"` value reaches the stub's `IsValidStructuralType` check uninterrupted, which is what the new test relies on. Defense-in-depth at multiple layers; the test honestly exercises the deepest enforcement layer (domain) plus the MCP-boundary error mapping (stub `errors.Join` → `invalid_request:` prefix).
