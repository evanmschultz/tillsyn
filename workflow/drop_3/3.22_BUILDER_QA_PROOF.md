### 3.2 Test 7 uses `repo.UpdateActionItem` to flip RequireChildrenComplete (non-blocking)

Lines 635-642 reach into the repo directly to set `RequireChildrenComplete=true` because the policy field is not exposed via the service-layer Update path pre-Drop-1. The inline comment at lines 628-634 documents the rationale and the future drop boundary ("pre-Drop-1 the policy is opt-in via metadata"). When Drop 1 lands the always-on default, this `repo.UpdateActionItem` call becomes redundant and should be deleted — leaving a TODO comment for that future cleanup would be nice but not necessary; the existing inline comment is sufficient.
