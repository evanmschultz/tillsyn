### 2.7 `irreducibleRaw int` non-canonical-bool overflow (LOW severity if hit)

**Hypothesis:** SQLite stores INTEGER as int64; `irreducibleRaw int` could overflow on a 32-bit GOARCH for large values (e.g., 2^33). Or non-canonical 0/1 values (e.g. 5) silently parse as `true`.

**Method:** Two responses:

(i) Overflow: Tillsyn requires no 32-bit GOARCH support per the project's Go 1.26+ toolchain assumption; on 64-bit `int` == int64 so no overflow.

(ii) Non-canonical: yes, `irreducibleRaw != 0` collapses any non-zero value to `true`. But the only writer in code is `boolToInt(bool) int` which returns exactly 0 or 1, and there is NO migration backfill (per 2.5). So no row can carry a non-canonical value through normal operation. The `!= 0` form is also more permissive than `== 1` would be — should a future bug write 2, the field still rounds to true rather than ignoring the truthy bit. Not a counterexample, this is a contract narrowing the team accepted by mirroring SQLite-bool conventions. REFUTED.
