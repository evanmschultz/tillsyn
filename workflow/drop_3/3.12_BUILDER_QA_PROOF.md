## 3. Findings

### 3.1 (Informational) JSON encoding shape uses Go field names verbatim, not snake_case (NIT — not blocking)

`KindCatalog` adds explicit `json` tags (`schema_version`, `kinds`, `agent_bindings`) on its top-level fields. The nested `KindRule` and `AgentBinding` types in `internal/templates/schema.go` carry only `toml` tags — no `json` tags. Go's default JSON encoder will use the exported field names verbatim (`Owner`, `AllowedParentKinds`, `MaxBudgetUSD`, etc.).

This is fine for the current droplet because:
- The catalog persists round-trip via `encoding/json` only (no cross-language consumer reads the JSON shape today).
- `TestKindCatalogJSONRoundTrip` proves marshal→unmarshal stability and `TestKindCatalogResolutionFromBakedCatalog` proves the integration round-trip end-to-end.
- Droplet 3.12's acceptance criteria do not pin the on-disk JSON casing.

If a future drop adds a non-Go consumer of `KindCatalogJSON` (a future TUI explorer, a debug dump, an external dispatcher), it may want to retrofit `json` struct tags onto `KindRule` / `AgentBinding` for snake_case symmetry with the TOML wire format. Capture this as a Drop 4+ refinement candidate, not a blocker.

### 3.2 (Informational) `Duration` JSON encoding is int64 nanoseconds

`templates.Duration` is `type Duration time.Duration` with no explicit `Marshal/UnmarshalJSON` methods. JSON encoding produces an integer nanosecond count (e.g. `30000000000` for 30s) rather than the canonical `"30s"` TOML string form. Round-trip is symmetric so persistence works, but the JSON-on-disk debug experience differs from the TOML source. Same Drop 4+ refinement bucket as 3.1.

### 3.3 (Confirmed) Soft-failure design on malformed JSON is correct

`lookupKindDefinitionFromCatalog` at `:625-630` swallows `json.Unmarshal` errors and returns `(zero, false, nil)`. The acceptance criterion explicitly mandates this behavior: a malformed envelope must not brick boot, and the legacy repo path covers compatibility. `TestKindCatalogResolutionFallsBackOnMalformedJSON` (`:125-152`) seeds `{not valid json` and asserts the fallback completes successfully. This is intentional, documented, and tested.

### 3.4 (Confirmed) Cycle trap explicitly avoided per fix L5

Verified `internal/domain/project.go` has zero `internal/templates` imports — only docstring references explaining *why* the typed catalog cannot live here. `Project` exposes no decoder method; all decoding happens in `internal/app/kind_capability.go:lookupKindDefinitionFromCatalog`. Fix L5 (CE4 import-cycle resolution) is honored.
