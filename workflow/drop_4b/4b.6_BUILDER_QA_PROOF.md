### 3. Falsification Attacks Survived

- **Attack 1 — Env-isolation list incomplete vs gitdiff round-3.** Mitigated: side-by-side `rg` of `git_status.go` vs `gitdiff/exec_differ.go` + `_test.go` confirms identical 7-key set (filter `GIT_*` + 4 isolation keys + 2 non-interactive keys). The `filteredGitEnv` helper at `git_status.go:135-145` mirrors `gitFixture.git`'s strip pattern at `exec_differ_test.go:156`.
- **Attack 2 — Pre-check fires AFTER repo write.** Mitigated: line-ordering inspection at `service.go:930-934` vs `:972` proves the pre-check is upstream of `s.repo.CreateActionItem`. Worklog §"Insertion-point ordering" pins the rationale.
- **Attack 3 — Empty `Paths` triggers needless git invocations.** Mitigated: `service.go:930` guards on `len(in.Paths) > 0`. `defaultGitStatusChecker:67-69` also guards internally. `TestCreateActionItemHandlesEmptyPaths` asserts `len(checker.calls) == 0` for the empty-paths path.
- **Attack 4 — Tests use stub fakes that bypass real `git status`.** Mitigated: 7 production-checker tests in `git_status_test.go` use real `t.TempDir()` + `git init` + commits via the `gitStatusFixture`. `TestCreateActionItemEnvIsolatesFromBareRoot` runs the production checker end-to-end through `Service.CreateActionItem` against a real fixture worktree. Stub-only would fail this test.
- **Attack 5 — `ErrPathsDirty` not a typed sentinel.** Mitigated: `errors.go:18` defines via `errors.New(...)`; wrapping uses `%w` at `service.go:1036`; `errors.Is(err, ErrPathsDirty)` is asserted in three tests.
- **Attack 6 — `mage test-pkg` race-flag missing.** Mitigated: `mage test-func` invocation logs `-race -count=1`; `mage test-pkg ./internal/app` succeeded in the same run with race detection on (mage's package-test default). `Test...IgnoresInheritedGitDir` and `Test...EnvIsolatesFromBareRoot` both correctly omit `t.Parallel()` because `t.Setenv` mandates serial execution — race detector wouldn't catch that, but the `t.Setenv` panic-on-parallel guard does. Stub-checker tests use `t.Parallel()` because they don't touch process env.
- **Attack 7 — Skip semantics swallow real misconfigurations.** Mitigated: project-lookup-failure skip is documented in `service.go:1018-1023` doc-comment as best-effort with the downstream repo write being authoritative source of "not found." Empty-worktree skip is the explicit pre-MVP escape valve from WAVE_C_PLAN §3.4 acceptance 2 — Drop 5 dogfood will validate that real projects always carry `RepoPrimaryWorktree`. Both branches produce zero false-positives in normal flow because Project bootstrap (4a.12) populates `RepoPrimaryWorktree` for every real project.
- **Attack 8 — Multiple-dirty-paths message non-deterministic.** Mitigated: `service.go:1036` uses `strings.Join(dirty, ", ")` over the input-order-preserved slice from `defaultGitStatusChecker:77-86` (`for _, path := range paths`). `TestCreateActionItemRejectsMultipleDirtyPaths` pins `alpha.go` AND `gamma.go` named, `beta.go` NOT named — deterministic + filtered.
- **Attack 9 — `--` argument ordering injection.** Mitigated: `git_status.go:99` uses `--` separator before `path` so a user-controlled path starting with `-` cannot masquerade as a flag. Also blocks refname collision (e.g. a path called `HEAD`).
- **Attack 10 — Builder helper landed in wrong package.** Spec/WAVE_C_PLAN §3.4 acceptance 1 suggested `internal/domain/git_status.go`. Builder placed it in `internal/app/git_status.go` per explicit spawn-prompt path scope (`internal/app/git_status.go (NEW)`). Worklog §"Helper lives in `internal/app`" documents the deliberate scoping decision. The function-type seam is internal to the package today; future cross-package use can promote it. **Not a violation** — spawn-prompt path list overrides the wave plan when they conflict, and the spawn list is what was authorized.

No unmitigated counterexamples surfaced.
